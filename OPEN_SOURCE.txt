ABOUT THIS SOFTWARE
--------------------------------------------------------------------------------
IV VoxNova (IP-A1 Control + TTS Paging System) is a free, open-source web
application for controlling TOA IP-A1 series network ceiling speakers and
sending Text-to-Speech announcements over a local area network (LAN).

It is designed for educational institutions, offices, and any environment using
TOA IP-A1 series speakers where teachers or staff need a simple, mobile-friendly
interface to control audio and make voice announcements without physical access
to amplifiers or dedicated paging hardware.

Key features:
  - Role-based user access (User / Admin / IT)
  - Text-to-Speech announcements via Kokoro TTS (local, offline, free)
  - Direct SIP audio routing to individual speakers
  - Paging Gateway (PG) routing for multicast zone paging
  - Multiple configurable paging gateways
  - Contact-based routing with per-user assignments
  - Full audit logging of all system activity
  - QR code for instant phone/tablet connection
  - TTS job queue for concurrent users

This software is NOT affiliated with, endorsed by, or officially supported by
TOA Corporation or TOA Canada. It is a third-party community tool that uses
TOA's publicly documented HTTP API.


--------------------------------------------------------------------------------
TOA SPEAKER API — THIRD-PARTY HARDWARE NOTES
--------------------------------------------------------------------------------
This software communicates with TOA IP-A1 series speakers using their built-in
HTTP REST API with HTTP Digest Authentication (RFC 2617).

The TOA IP-A1 API documentation and technical manuals are available from:

  TOA Corporation (Japan):
    https://www.toa.co.jp/en/

  TOA Canada Corporation:
    https://www.toacanada.com/

  TOA Electronics (USA):
    https://www.toaelectronics.com/

Relevant product series for which this software is designed:
  - IP-A1S (ceiling speaker, PoE)
  - IP-A1F (flush-mount ceiling speaker)
  - IP-A1T (tile-type ceiling speaker)
  - Other IP-A1 series models with HTTP API v2 support
  - IP-A1PG (Paging Gateway for multicast zone audio)

To obtain the full IP-A1 API specification, contact TOA technical support or
refer to the IP-A1 series installation and technical manual, which documents:
  - HTTP Digest Authentication requirements
  - All available API endpoints (/api/v2/...)
  - Volume control commands
  - Mute control commands
  - System information endpoints
  - Firmware and model information endpoints

WHAT THIS SOFTWARE DOES WITH THE TOA API:
  This application acts as an HTTP proxy. The Node.js server receives requests
  from browsers (which cannot perform Digest Authentication natively) and
  forwards them to the speaker hardware with proper Digest Auth headers.
  No data is modified in transit. No data is sent to any external server.
  All communication stays within the local network.

API ENDPOINTS USED (base URL: http://<speaker_ip>):
  GET /api/v2/volume/get_master            — Read current volume
  GET /api/v2/volume/set_master?volume=N   — Set volume (N = 0 to 61)
  GET /api/v2/volume/inc_master            — Increment volume
  GET /api/v2/volume/dec_master            — Decrement volume
  GET /api/v2/volume/get_master_mute       — Read mute state
  GET /api/v2/volume/set_master_mute?mute_state=mute   — Mute
  GET /api/v2/volume/set_master_mute?mute_state=unmute — Unmute
  GET /api/v2/firmware/model               — Read model information

THIRD-PARTY SOFTWARE & SYSTEM REQUIREMENTS
--------------------------------------------------------------------------------
IV VoxNova requires the following third-party software to be installed on the
host machine. All are free and open-source unless otherwise noted.

  Node.js 18+                MIT     JavaScript runtime
    https://nodejs.org
    Required to run the IV VoxNova server.

  Python 3.12                PSF     Python programming language
    https://www.python.org/downloads/release/python-3120/
    IMPORTANT: Must be installed with "Add Python to PATH" checked.
    Required for the Kokoro TTS engine. Python 3.12 is specifically
    recommended for compatibility with the kokoro package.

  kokoro (Python package)    Apache 2.0   TTS model engine
    Install: pip install kokoro
    An 82-million parameter, offline text-to-speech model.
    Runs entirely locally — no internet connection required during use.

  soundfile (Python package) BSD     Audio file I/O library
    Install: pip install soundfile
    Required by kokoro for writing WAV audio output.

  ffmpeg                     LGPL    Audio/video processing
    https://ffmpeg.org/download.html
    Must be in system PATH (add ffmpeg/bin to PATH after extracting).
    Required for encoding audio and delivering it over SIP/RTP to speakers.

  To install all Python packages at once:
    pip install kokoro soundfile

Contributions are welcome. If you improve this software, please consider

When contributing:
  - Keep the code readable and well-commented
  - Test on both desktop browsers and mobile (iOS Safari, Android Chrome)
  - Ensure changes work with the production build (start.bat), not just dev mode
  - Do not hardcode credentials or site-specific configuration

Areas where contributions are especially valuable:
  - Support for additional TOA speaker models or firmware versions
  - Accessibility improvements
  - Additional language translations for the UI
  - Better offline/PWA support (service worker caching)
  - Support for additional TTS engines or languages
  - Volume scheduling (set volume automatically at certain times)
  - SIP PBX integration for enterprise deployments

SECURITY NOTES
--------------------------------------------------------------------------------
This software is designed for CLOSED LAN use only.

  - Speaker passwords are stored in plain text in rooms.json
  - User passwords are stored as bcrypt hashes in users.json
  - JWT tokens are signed with a secret stored in the server process
  - There is no HTTPS / TLS encryption (plain HTTP over LAN)
  - No rate limiting or brute-force protection is implemented

These are acceptable trade-offs for an internal network tool.

DO NOT:
  - Expose this server to the public internet
  - Use it over an untrusted VPN connection
  - Store sensitive credentials beyond speaker login credentials

If your site requires higher security:
  - Place the server behind a reverse proxy with HTTPS (e.g. nginx + Let's Encrypt)
  - Set a strong JWT secret via environment variable
  - Change default admin/IT passwords immediately after installation
