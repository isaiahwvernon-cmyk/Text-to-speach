PROJECT: REPIT – IP-A1 Control + TTS Paging System

Overview
Build a standalone web-based control application that allows users to:
- Control volume of TOA IP-A1 speakers
- Send live Text-to-Speech (TTS) announcements
- Route audio either Direct (peer-to-peer SIP) or via IP-A1PG (SIP → DTMF → Multicast zones)

CORE FEATURES

1. Volume Control
- Per speaker or zone control
- Permission-based visibility

2. Text-to-Speech Paging
- Uses Kokoro TTS (server-side)
- Supports G.711 μ-law (PCMU), G.711 A-law (PCMA), G.722

3. Routing Modes

A) Direct Mode
User → Server → TTS → Transcode → SIP → RTP → Speaker

B) PG Mode
User → Server → TTS → Transcode → SIP to PG → DTMF → Delay → (Optional Chime) → Delay → RTP → PG → Multicast → Speakers

DELAY LOGIC
- DTMF Delay: 500–750 ms (adjustable)
- Chime Delay: 500–1000 ms (adjustable)

PERMISSIONS
